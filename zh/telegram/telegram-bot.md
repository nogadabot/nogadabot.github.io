# Telegram 机器人使用指南（完整版）

Nogada Telegram 机器人即使**关闭应用程序也能 24 小时**持续铸造和管理钱包。核心功能是**机器人任务**：只需在 Telegram 中发送**一行指令**，**服务器就会直接**用您的机器人钱包进行铸造。即使电脑关机、您在睡觉，它也照样运行。

> 🤖 机器人地址：**@NOGADA\_Mint\_Bot**

> 💡 **机器人任务 vs 应用任务：先了解这一点**
> * **🤖 机器人任务（24/7）**：服务器**直接用机器人钱包**铸造。**应用程序（电脑）可以关闭。** 这是本文档的核心。
> * **💻 自动铸造（应用任务）**：在应用程序中创建并运行任务（支持 OpenSea 中继·完整高级功能）。但**应用程序必须保持开启**。
> 如果您想在睡觉时也能抢到铸造机会 → 请使用**机器人任务**。

## 1) 连接（配对）

机器人只有**与您的许可证绑定的人**才能控制。

1. 在应用程序中前往**设置 → 集成（Integrations）**。
2. 查看那里显示的**配对码**。
3. 在 Telegram 中打开 **@NOGADA\_Mint\_Bot**，将该**配对码原样发送**过去。
4. 收到"连接成功！"消息后即完成。

![设置 - 集成（Telegram 配对）](../images/settings-integrations.png)

> *设置 → 集成。**Telegram** 区块会显示连接状态。如果尚未连接，这里会显示**配对码**，将该码发送给机器人即可完成连接。*

> ⚠️ **请像对待密码一样保管配对码。** 录屏或直播时切勿暴露。看到配对码的人可以连接您的机器人并盗取机器人钱包中的资产。

> 💡 一个许可证 = 一个已连接的聊天。在新设备/账号上重新连接后，旧连接会自动解除。

## 2) 机器人钱包（服务器托管）：铸造所需资金

通过 Telegram 铸造需要**机器人钱包**中有余额。机器人钱包与应用程序内的钱包不同，它**（加密后）保存在服务器上**，即使应用程序关闭，机器人也能代为签名和发起交易。

**菜单：💼 钱包 → 🤖 机器人钱包（24/7）**

| 按钮 | 功能 |
|---|---|
| ➕ 创建钱包 / ➕ 创建 5 个 | 生成新的机器人钱包（地址） |
| 🔐 导入私钥 | 将现有私钥（0x+64位）注册为机器人钱包 |
| 🔑 导出私钥 | 获取机器人钱包私钥（录屏时禁止·收到后删除消息） |
| 💸 提取 | 将机器人钱包中的 ETH/资金**转入您的主钱包** |
| 🗑 删除 | 删除机器人钱包 |

> 🔐 **机器人钱包仅供"一次性（小额）"使用。请勿存入大量资金。**
> 为了实现 24 小时自动运行，私钥需托管在服务器上，因此原则上**只存入铸造所需的金额**。主要资金请存放在应用程序的**应用钱包**中（仅保存在本机）。铸造获得的 NFT 或剩余资金随时可通过**💸 提取**转移到安全钱包。

## 4) ⭐ 机器人任务：一行指令完成铸造

这是核心功能。点击**⚙️ 任务 → 🤖 机器人任务 → ➕ 新建机器人任务**，机器人会提示您一行格式。按照以下方式发送后即可创建任务，点击**▶**后所有机器人钱包会同时发起铸造。

### 基本格式

```
0x合约地址 [链] [数量] [价格ETH] [模式]
```

| 参数 | 说明 | 省略时默认值 |
|---|---|---|
| **0x合约地址** *（必填）* | 要铸造的 NFT 合约地址 | — |
| 链 | eth · base · arb · op · poly · blast · linea · scroll · zora · avax · bnb · abstract · ape · ink · sepolia | `ethereum` |
| 数量 | 每个钱包铸造数量（1~100） | `1` |
| 价格ETH | 每次发送的 ETH（付费铸造时填写价格） | `0` |
| 模式 | `auto`=`mint(数量)` · `sea`=OpenSea SeaDrop · `0x…`=直接 calldata | `auto` |

> 最简示例：`0xed5af3...c544` → 在以太坊上对该合约调用 `mint(1)`，免费 1 次。
> 付费 1 个：`0xed5af3...c544 eth 1 0.01 sea` → 通过 OpenSea SeaDrop 以 0.01 ETH 铸造 1 个。

### 选项：在后面空格追加 `选项=值`

| 选项 | 说明 | 示例 |
|---|---|---|
| `fire=spam` | 🔥 **连发**：持续重试直到成功，**首次成功后自动停止** | `fire=spam` |
| `fire=safe` | 🛡 **安全**：模拟后仅执行 1 次（跳过会失败的钱包） | `fire=safe` |
| *（无）* | **立即 1 次**（默认） | |
| `tip=` | ⛽ 优先费（gwei），**越高打包越快** | `tip=8` |
| `fee=` | ⛽ 最大费用（gwei）上限 | `fee=80` |
| `gas=` | Gas 上限（手动指定） | `gas=120000` |
| `max=` | 连发**最大发送次数**（留空则无限制） | `max=200` |
| `delay=` | 连发**间隔（ms）** | `delay=300` |
| `at=` | ⏰ **预约发射**：`+5m`·`+2h`·`HH:MM`（UTC） | `at=16:59` |
| `sig=` | 🔧 铸造**函数签名** | `sig=mintPublic(uint256,address)` |
| `args=` | 🔧 函数参数（用 `;` 分隔，可使用 `{address}`） | `args=2;{address}` |
| `sweep=` | 📦 铸造后自动转移 NFT 的**安全钱包** | `sweep=0xSafe…` |
| `fb=1` | 🛡 主网 **Flashbots**（立即/安全模式） | `fb=1` |

顺序不限。示例：`0xed5af3...c544 eth 1 0.01 fire=spam tip=8 at=+10m`

---

### 🔘 铸造模式（`fire=`）

| 模式 | 说明 | 适用场景 |
|---|---|---|
| **立即 1 次** *（默认）* | 每个钱包**立即发射 1 次** | 在铸造时间直接点击 ▶ 或**预约**时。速度最快 |
| `fire=safe` | 先**模拟运行** → 跳过会失败的钱包，通过的执行 1 次 | 不浪费 Gas，安全铸造 |
| `fire=spam` | 持续重试直到成功 → 成功后自动停止 | 不确定铸造时间或竞争激烈时 |

> 💡 **连发（`fire=spam`）是"开启后自动抢单"的方式。** 铸造未开放时持续尝试，开放瞬间抓住铸造机会，**链上确认后自动停止**（不再消耗 Gas）。

> ⚠️ **安全（`fire=safe`）不是"重试"。** 它会模拟运行一次，然后发射 1 次就结束。如果在**铸造尚未开放时**启动 → 模拟失败 → 直接结束。如果想提前启动，请使用**预约（`at=`）**或**连发（`fire=spam`）**。

### ⏰ 预约发射（`at=`）

提前创建并点击 ▶，机器人会**等到指定时间后精确发射**（期间显示为"已武装"状态）。

| 用法 | 说明 |
|---|---|
| `at=+5m` · `at=+2h` · `at=+30s` | **从现在起** 5 分钟·2 小时·30 秒后 *（不易混淆，推荐）* |
| `at=16:59` | 今天/明天 **16:59（UTC 时间）** |

> 🕘 **`HH:MM` 以 UTC（世界标准时间）为准。** 中国时间（CST）为 UTC+8。例）中国时间下午 5:00 铸造 = **`at=09:00`**（UTC）。如果容易混淆，建议使用**相对时间（`+5m`/`+2h`）**。

> ⚠️ **预约在服务器重启后会丢失。** 运营者重新部署/重启服务器后，正在等待的预约任务会被取消（较少发生）。届时请重新点击 ▶。

### ⛽ 手动指定 Gas（`tip=` `fee=` `gas=`）

默认为**自动**（根据市场自动调节）。在竞争激烈的铸造中**想要明显领先**时，请手动调高。

* `tip=` **优先费（gwei）**：**这是竞争的关键。** 给矿工的小费，越高您的交易越会**优先**被打包。
* `fee=` **最大费用（gwei）**：单笔交易愿意支付的 Gas 上限。
* `gas=` **Gas 上限**：通常使用自动（估算）就足够。仅在估算失败的特殊合约中手动指定。

> 💡 在热门 Drop 中频繁落选 → **请提高 `tip=`**（例如 `tip=15`）。Gas 出得越多，在竞争中越占优势。

### 🔁 连发调节（`max=` `delay=`）

仅在连发（`fire=spam`）时有意义。

* `delay=` **发送间隔（ms）**。`300` = 每 0.3 秒一次，`1000` = 每 1 秒一次。越小越激进，越大越节省 RPC·Gas。
* `max=` **最大发送次数**（上限）。⚠️ 不是"成功 N 个"，而是**发送 N 次后停止**。留空则无限制。对于**已售罄/不符合条件等永久无法铸造**的项目，请设置上限以避免无限重试（或手动点击 ⏹ 停止）。

### 🔧 自定义函数（`sig=` `args=`）

当需要调用**非 `mint(数量)` 的其他铸造函数**时使用。

* `sig=mintPublic(uint256,address)`，函数名 + 参数类型。
* `args=2;{address}`，各值用**`;`分隔**。`{address}` 会**自动替换为各机器人钱包地址**。
* 如果函数**需要付费（payable）**，发送的 ETH 填写在前面的**价格ETH**参数中。

> 示例：`0xabc… eth 1 0.03 sig=mintPublic(uint256,address) args=1;{address}`
> → 以 0.03 ETH 调用 `mintPublic(1, 我的机器人钱包地址)`。

> 💡 如果觉得复杂，不用也没关系。大多数情况下用默认的 `auto`（=`mint(数量)`）或 OpenSea 的 `sea` 就够了。**领取运营者发布的 🎁 Drop** 时，这些设置**已经全部配置好**了。

### 📦 铸造后自动转移（`sweep=`）

添加 `sweep=0x安全钱包` 后，**铸造成功后立即**将收到的 NFT（ERC-721）**自动发送**到该安全钱包。适合不想让 NFT 留在临时机器人钱包、想直接转入主钱包的场景。

> 📦 ETH（剩余资金）不会自动转移，需要通过**💸 提取**手动操作。

### 🛡 Flashbots：防止主网抢跑（`fb=1`）

添加 `fb=1`（仅限以太坊**主网**）后，交易不经过公开内存池，而是**直接私密发送给矿工**。

* **为什么？** 将有价值的铸造交易发布到公开内存池，**MEV 机器人可能看到后加高 Gas 抢先执行**。Flashbots 是防止这种情况的**秘密通道**。
* **仅在立即/安全模式下**有效（连发模式是通过公开方式快速重试，因此不支持）。
* **无公开回退**：如果 bundle 未被打包进区块，交易就直接消失（不会再次暴露到公开内存池）。失败后可以重试。

> 平时关闭，在**竞争激烈的主网热门 Drop** 中开启以防止抢跑。示例：`0xabc… eth 1 0.08 tip=15 fb=1`

### 🎬 实战示例合集

```text
# ① 10 分钟后立即发射 1 个（OpenSea 付费 Drop）
0xed5af3...c544 eth 1 0.05 sea at=+10m

# ② 竞争铸造 — 连发抢单（中国时间下午 5 点 = UTC 09:00）
0xed5af3...c544 eth 1 0.01 fire=spam tip=10 delay=300 at=09:00

# ③ 安全模式 — 仅模拟通过的钱包（Base，2 个）
0xed5af3...c544 base 2 0.02 fire=safe

# ④ 自定义函数 + 铸造后转移到安全钱包
0xabc... eth 1 0.03 sig=mintPublic(uint256,address) args=1;{address} sweep=0xSafe...

# ⑤ 主网 — Flashbots 私密模式（防止抢跑）
0xabc... eth 1 0.08 tip=15 fb=1
```

> 点击 ▶ 后，**所有已连接的机器人钱包同时（并行）**发起交易。结束后 Telegram 会发送**每个钱包的结果**（成功/失败·获得数量·转移数量·tx 链接）。

## 5) 🎁 领取 Drop：24/7 领取运营者 Drop

运营者（管理员）发布 Drop 后，在机器人菜单的 **🎁 Drop** 中可以查看列表。

1. 打开 **🎁 Drop** → 查看每个 Drop 的**阶段**（Public/GTD/FCFS 等）·价格·最大数量。
2. 点击想要领取的阶段的**「领取」**按钮。
3. 该阶段的配置会**自动创建为机器人任务**（合约·函数·价格·数量全部已配置好）。
4. 点击**▶**执行：或如果设置为**连发**，会自动重试直到铸造开放。

> 💡 无需手动输入合约地址·函数，这是**一键领取运营者已配置好的内容**的最简便方式。机器人钱包有余额即可。

> *仅有 OpenSea **slug**（无合约地址）的 Drop，以及部分需要白名单签名的阶段会显示为"在应用中领取"：请通过应用程序的**🤖 自动铸造**领取。*

## 8) 安全与技巧

* 🔐 **机器人钱包只存小额。** 大额资金放应用钱包（本机）。铸造后通过 **💸 提取**或 `sweep=` 转出。
* 🤐 **配对码·导出的私钥请像密码一样保管。** 直播·录屏时禁止暴露。
* ⛽ **热门 Drop 请提高 `tip=`**，并设置快速的 **RPC**。
* ⏰ **不知道时间就用连发**（`fire=spam`），**知道时间就用预约**（`at=`）。两者可以结合使用。
* 🕘 `HH:MM` 是 **UTC** 时间。中国时间为 UTC+8，容易混淆时请用 `+5m`/`+2h`。
* 🛑 永久无法铸造的项目（已售罄·不符合条件）请用 `max=` 设置上限，或点击 ⏹ 停止，避免无限重试。
