# 任务 (Tasks)：铸造的核心

**任务**是一条铸造配置，记录了"用哪个合约、用哪个钱包、以何种方式铸造"。创建任务并点击**执行 (Run)** 后，铸造即开始。

> ⭐ **对铸造方式（即时·安全·连发）、延迟、防护机制、预约启动、多钱包（并行）感到困惑，请先阅读 → [铸造方式完全指南](../minting/modes.md)**。本页侧重界面位置，那页侧重"何时用什么"。

![tasks](../images/tasks.png)

## 界面说明

* **任务组（左侧）**：将任务分组管理。点击 `+ 新建组` 创建组。（例如："白名单铸造"、"公开"等）
* **工具栏（右上角）**
  * **+ 新建任务**：打开任务编辑器（详见下方）。
  * **快速发射**：对选中的任务快速发射。
  * **组启动 / 组停止**：一次性执行/停止组内所有任务。
  * **批量编辑**：一次性修改多个任务的设置。

> ⚠️ 任务必须放在**组内**。若还没有任何组，请先点击 `+ 新建组`。

## ⭐ 创建新任务（编辑器完整说明）

点击 `+ 新建任务` 即可打开编辑器窗口。下面逐一说明各个字段。

![task editor](../images/task-editor.png)

![ABI 导入放大图](../images/task-editor-detail.png)

> 🔍 *放大图：输入合约地址后点击 **导入 ABI**，函数将自动填充。*

### 🎯 实战示例：跟着做第一次铸造

**目标：** 用 **2 个钱包** 铸造 **1 个 0.05 ETH 的 NFT**。按编号顺序填写即可：

![实战示例](../images/task-example.png)

| # | 字段 | 做什么（示例值） |
|---|---|---|
| ① | **合约地址** | 粘贴要铸造的 NFT 合约地址，例如：`0x4E1f…480e56` |
| ② | **导入 ABI → 函数** | 点击**导入 ABI** → 应用读取合约并显示函数列表。选择 **`mint(uint256)`**，函数字段即自动填充。 |
| ③ | **参数 (Arguments)** | 选择函数后，**参数字段会自动生成**（显示名称和类型）。`mint(uint256)` 的话，**数量字段 → 填 `1`** |
| ④ | **金额 (ETH)** | 每个的**铸造价格 → `0.05`**（免费铸造填 `0`） |
| ⑤ | **钱包 + RPC** | 勾选用于铸造的钱包（**全选** = 2 个），至少勾选 1 个 RPC |
| ⑥ | **创建** | 完成：任务添加到列表，准备好**执行 (Run)** |

> 💡 红色的 *"钱包余额低于预计执行费用"* 警告，只是表示该钱包目前还不足 0.05 ETH。请先充值 → [资金管理](../app-guide/wallets.md)。

> 🧩 **不知道函数或参数？** 这正是 **② 导入 ABI** 的用途——它会显示合约的实际函数，无需猜测。大多数公开铸造用 `mint(uint256)` 并将数量作为参数即可。

### 左侧："铸造什么"

| 字段 | 填写内容 |
|---|---|
| **任务名称** | 留空则自动命名。如需区分可手动输入。 |
| **目标（选项卡）** | 从**合约** / **OpenSea 铸造** / **发射台** 中选择（说明见下） |
| **合约地址** | 要铸造的 NFT 合约地址（`0x...`） |
| **链** | 铸造的目标链（Ethereum、Base 等） |
| **原始 calldata** | 勾选后，直接输入 **hex 原始数据**而非函数（高级用法） |
| **函数** | 铸造函数（例如：`mint(uint256)`）。可用旁边的**导入 ABI** 自动填充 |
| **参数 (Arguments)** | **从 ABI 选择函数后，参数字段会按名称和类型逐项自动生成**：无需猜测顺序。`{address}` = 各钱包地址，`{id}` = 代币 id，自动替换。若函数**需要付费 (payable)**，金额字段为**必填**（红色星号）。 _(不使用 ABI 直接输入时，用 `;` 分隔)_ |
| **金额 (ETH)** | 每个的铸造价格。免费则填 `0` |

#### 🔧 "导入 ABI"：无需手动输入函数

输入合约地址后点击**导入 ABI**，将自动拉取该合约的**函数列表**，以下拉菜单形式展示。从中选择铸造函数（`mint`、`publicMint` 等），函数字段即自动填充。（在[设置](../app-guide/settings.md)中填入 Etherscan API 密钥效果更佳；没有也会尝试通过 Sourcify 获取。）

> 💡 **目标选项卡区别**
> * **合约**：直接通过合约地址 + 函数铸造（最常见）
> * **OpenSea 铸造 / 发射台**：只需输入铸造链接，即可**自动识别铸造阶段 (phase)**。选择阶段后填写其余信息即可。（支持：OpenSea、Transient）

### 右侧："如何铸造"

| 字段 | 填写内容 |
|---|---|
| **副本数 (Copies)** | 相同任务重复执行的次数 |
| **钱包 (Wallets)** | 勾选用于铸造的钱包。**全选**可选中所有钱包 |
| **RPC** | 勾选要使用的 RPC 端点。（不选则使用公共 RPC 执行） |

### 底部：保存

* **模板**：将常用设置保存为模板或从中加载。
* **取消 / 创建**：点击**创建**，任务即创建完成。

## 📋 新建任务窗口：所有字段说明（一览）

> 整理了新建任务弹窗中出现的**所有字段**。关于概念（何时用什么），请参阅 → [铸造方式完全指南](../minting/modes.md)。

### 🎯 目标："铸造什么？"
| 字段 | 说明 |
|---|---|
| **目标选项卡** | **合约**（通过地址直接铸造）/ **OpenSea 铸造**（链接）/ **发射台** |
| **合约地址 / 铸造链接** | 要铸造的 NFT 的 `0x…` 地址，或 OpenSea/发射台的**链接** |
| **链** | 铸造的目标区块链（Ethereum、Base 等） |
| **原始 calldata** _(高级)_ | 直接输入 **hex 原始数据**而非函数。一般不使用 |
| **函数** | 铸造函数（例如：`mint`）。使用旁边的**导入 ABI** 自动填充 |
| **参数** | 函数所需的值。**选择函数后，字段按名称和类型自动生成**。`{钱包}`/`{代币id}` 自动替换 |
| **金额 (ETH)** | 每个的铸造价格。免费填 `0`。**需要付费 (payable) 的函数为必填**（红色星号） |

### 🔥 铸造方式（三选一）
| 方式 | 说明 |
|---|---|
| **即时 1 次** | 每个钱包**发射一次**，速度最快（推荐配合预约使用） |
| **安全 1 次** | **模拟执行**后，跳过预计失败的钱包，发射 1 次（防止浪费 Gas） |
| **连发** | **不断重试直到成功** → 成功后**自动停止**（挂着让它自动抢） |

### ⛽ Gas（手续费）
| 字段 | 说明 |
|---|---|
| **Gas 限额** | 交易可使用的 **最大 Gas**。一般留空自动处理 |
| **最大费用** | 每单位 Gas 愿意支付的**最大值 (gwei)**。自动 = 跟随市场 |
| **优先费** | **矿工小费 (gwei)**。越高越优先上链，竞争中更占优势 |

### ⏱️ 时机与重试
| 字段 | 说明 |
|---|---|
| **启动时间** | **预约发射时间**（日历+时钟，本地时间）。留空则立即发射 |
| **延迟** _(连发)_ | 重试间隔。`100` = 每 0.1 秒一次，`0` = 最大速度 |
| **最大 tx 数** _(连发)_ | 最大发送次数。留空则无限制（直到成功或手动停止） |
| **nonce** _(非连发时)_ | 发送交易的序号。一般自动处理 |

### 👛 钱包与 RPC
| 字段 | 说明 |
|---|---|
| **钱包** | 用于铸造的钱包（可多选）。**默认每个钱包创建 1 个任务**（并行）。也可选择"合并到一个任务" |
| **RPC** | 使用的节点。不选则使用公共 RPC 执行 |

### ⚙️ 选项
| 字段 | 说明 |
|---|---|
| **铸造后操作** | 铸造成功后自动执行**转账 / OpenSea 上架 / 接受报价** |
| **Flashbots 捆绑** | 通过**私有内存池**提交（仅限以太坊），不在公开内存池暴露。有助于防止三明治攻击/抢跑 |
| **副本数** | 创建 N 个相同任务 |
| **模板** | 保存/加载常用执行设置 |

## ▶ 发射：执行 · 快速发射 · 停止 · 提速

创建任务后，实际发射的方法：

* **执行 (Run)**：发射**此单个任务**。在任务设定的启动时间发送交易（启动时间为`当前`时立即发射）。
  *示例：将白名单铸造的启动时间设为开放时间，点击**执行**后，一到时间自动发射。*
* **⚡ 快速发射**：最快的方式。**勾选**任务后点击**快速发射**，无需打开编辑器，**立即同时**全部发射。
  *示例：突然出现隐秘铸造 → 勾选已保存的任务 → **快速发射** → 立即发射。*
* **停止 (Stop)**：停止正在执行的任务。
* **🚀 提速 (Boost)**：当 Gas 过低导致交易卡住时，**以更高 Gas 重新发射**。详情 → [交易提速](../minting/boost.md)
* **组启动 / 组停止**：**一次性**执行/停止组内**所有任务**（适合同时多钱包、多合约发射）。

> 💡 **最佳实践**：在铸造时间前 3～5 分钟，**先停止任务再重新执行**，以加载最新链上数据（项目方有时会在最后一刻更改设置）。

Gas 设置不清楚？→ [Gas 设置完全指南](../minting/gas.md)
