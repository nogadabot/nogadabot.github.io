# 白名单（WL）铸造

白名单铸造是**仅限已登记钱包**才能铸造的阶段。通常需要证明（proof）或签名（signature），比公开铸造多一道准备步骤。

> 💡 铸造方式（即时·安全·连发）、预约开始、多钱包处理 → [铸造方式完全指南](modes.md)

## 最简单的方式：OpenSea / Transient 白名单

如果是 OpenSea·Transient 发射台的白名单，Nogada 会**自动处理 proof**。

1. **任务** → `+ 新建任务` → 目标选 **OpenSea 铸造** 或 **发射台**。
2. 粘贴铸造**链接**后，阶段将自动识别 → 选择 **白名单（allowlist）阶段**。
3. 选择**已加入白名单的钱包**。（可提前在 [钱包] 界面的**白名单确认**功能中检查钱包是否在白名单内。）
4. 填写 RPC·Gas，点击**创建** → **运行**。

## OpenSea 白名单获取方式：API 密钥 vs 默认

OpenSea SeaDrop **白名单（签名/allowlist）阶段**需要为每个钱包获取铸造 voucher 才能发射。获取途径有两种（在智能铸造·任务编辑界面的 **WL 获取方式** 中选择）：

* **OpenSea API（推荐）** — 使用本人的 **OpenSea API 密钥**获取 voucher。限额以**密钥为单位而非 IP**（每个密钥每分钟约 120 次 ≈ 约 50 个钱包），因此**无需代理**，钱包较多时增加密钥即可。密钥在**设置 → OpenSea API 密钥**中填入 — 在 `https://opensea.io/settings/developer` 创建，公司/网站栏填写任意内容即可。
  * 钱包超过 50 个时，请**单独创建 OpenSea 账号，每个账号添加一个密钥**。限额以**账号为单位**，同一账号下创建多个密钥不会叠加限额。
  * 密钥**30 天后过期**。过期后请重新创建并替换。
* **默认（代理）** — 无需密钥，使用本人的**代理 IP** 获取。限额以 IP 为单位，钱包较多时建议使用**多个代理**。

> 💡 **公开铸造两者都不需要。** 公开铸造无需 voucher 可直接铸造，密钥和代理均无关。此选项**仅在 WL/签名阶段**出现。

> 已登记密钥时，**OpenSea API 为默认选择**，创建任务时可切换为**默认（代理）**。

## 通用步骤（所有白名单铸造适用）

1. **确认应用为最新版本**（左下角版本号）。
2. **创建任务并提前查看日志**：点击运行（Run）后检查日志信息是否正常。
3. **铸造前 3～5 分钟停止后重新运行**：部分项目会在最后一刻修改配置，重新运行可加载最新数据。
4. **准备好手动铸造作为备份**：若铸造开始后 5～10 秒内任务日志未变为"等待发送"，请在项目官网手动铸造。

> 💡 **确认白名单钱包**：在 [钱包] 界面 → 选择钱包 → 使用**白名单确认**功能，可提前查看哪些钱包已进入该铸造的白名单。

## 自定义合约白名单（高级）

非发射台的自有合约白名单，通常**每个钱包的签名/proof 各不相同**。此情况下：

* 若运营者提供了单独说明（模块/hex），请按说明操作。
* 或者需要自行获取正确的 proof 并填入参数/hex 中（难度较高）。

> ⚠️ 白名单铸造中最常见的错误是**使用未在白名单内的钱包尝试铸造**或 **proof 填写错误**。运行前务必通过日志确认。
