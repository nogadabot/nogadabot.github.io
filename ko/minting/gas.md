# 가스 설정 완전정복

가스 설정은 민팅 **성공률과 비용**을 가르는 핵심입니다. 개념만 잡으면 어렵지 않습니다.

## 가스의 두 가지 모드

### 🔄 자동 (Auto)

앱이 **시장 상황에 맞춰 가스를 자동으로 올려줍니다.** (대기 중인 거래들을 보고 우선순위 수수료를 약간 더 높게)

* **언제**: 공급이 넉넉하거나 덜 치열한 민트, 급하지 않은 민트.
* 장점: 신경 안 써도 됨. 단점: 가끔 필요보다 조금 비쌀 수 있음.

### 🎯 직접 (Custom)

**최대 수수료(Max)**와 **우선순위 수수료(Priority)**를 직접 지정합니다.

* **언제**: hype 민트, 저공급, 가스 워, 고수익 기대 민트.
* 값은 본인이 판단합니다 — [도구의 가스 계산기](../app-guide/tools.md)로 비용을 미리 보고 정하세요.
* **느린/보장 민트**라면 Max는 적당히, **Priority는 0.1**처럼 낮게 = 최저가 민팅.

> Nogada에서는 [설정 → 엔진](../app-guide/settings.md)에서 **자동 팁 배수**와 **최소 priority(gwei)**의 기본값을 정하고, 작업별로 조정합니다.

## ⚠️ 가스 한도(Gas Limit)는 비워두세요

* 가스 한도는 "이 거래에 들어갈 작업량 상한"입니다. **낮춰도 수수료가 절약되지 않습니다.**
* **비워두거나 0**으로 두면 앱이 안전하게 자동 추정합니다.
* 직접 낮게 넣으면 **"out of gas"로 실패**해서 **가스비만 날립니다.** 전문가가 아니면 건드리지 마세요.

## 잔액은 넉넉하게

높은 커스텀 가스를 쓸 땐 **예상 비용보다 20~50% 많은 ETH**를 지갑에 두세요. 복잡한 컨트랙트는 가스 한도가 크게 잡혀 더 많은 ETH가 필요할 수 있고, 부족하면 트랜잭션이 아예 안 나갑니다.

## Flashbots (고급)

[설정 → 엔진](../app-guide/settings.md)에서 Flashbots 번들을 켜면, 트랜잭션을 공개 멤풀 대신 블록빌더에게 직접 제출할 수 있습니다 (프론트런 방지 등). 잘 모르면 기본값으로 두세요.

## 정답이 없는 영역

저공급 선착순(100~300개, 1~2블록 마감)은 **모든 사람의 가스·RPC 속도·인터넷·프로젝트 API 상태**가 얽혀서, "이 값을 넣으면 무조건 성공"이라는 정답이 없습니다. 몇 밀리초 차이로 갈리기도 합니다. 경험을 쌓으며 본인만의 감을 만드세요.
