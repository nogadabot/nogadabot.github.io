# 화이트리스트(WL) 민트

WL 민트는 **등록된 지갑만** 민팅할 수 있는 단계입니다. 보통 증명(proof)이나 서명(signature)이 필요해서 퍼블릭보다 한 단계 더 챙길 게 있습니다.

> 💡 민트 방식(즉시·안전·연사)·예약 시작·여러 지갑 처리는 → [민트 방식 완전정복](modes.md)

## 가장 쉬운 길: OpenSea / Transient WL

OpenSea·Transient 런치패드의 WL이라면 Nogada가 **proof를 자동으로 처리**합니다.

1. **작업** → `+ 새 작업` → 대상 **OpenSea 드롭** 또는 **런치패드**.
2. 민팅 **링크**를 붙여넣으면 단계가 자동 인식됩니다 → **WL(allowlist) 단계**를 선택.
3. **WL에 등록된 지갑**을 선택합니다. (지갑이 WL에 들어있는지는 [지갑] 화면의 **화이트리스트 확인**으로 미리 점검하세요.)
4. RPC·가스 채우고 **생성** → **실행**.

## OpenSea WL 받는 방식: API 키 vs 기본

OpenSea SeaDrop **WL(서명/allowlist) 드롭**은, 각 지갑의 민트 voucher를 받아와야 발사합니다. 받는 경로가 두 가지입니다 (스마트 민트·작업 편집 화면의 **WL 받기 방식**에서 선택):

* **OpenSea API (권장)** — 본인의 **OpenSea API 키**로 voucher를 받습니다. 한도가 **IP가 아니라 키 기준**(키 1개당 분당 약 120건 ≈ 지갑 약 50개)이라 **프록시가 필요 없고**, 지갑이 많으면 키를 더 추가하면 됩니다. 키는 **설정 → OpenSea API 키**에 넣습니다 — `https://opensea.io/settings/developer` 에서 만들고, 회사/사이트 칸은 아무거나 채우면 됩니다.
  * 지갑이 50개를 넘으면 **OpenSea 계정을 따로 만들어 계정마다 키 1개씩** 추가하세요. 한도는 **계정 단위**라, 같은 계정에서 키를 여러 개 만들어도 한도가 늘어나지 않습니다.
  * 키는 **30일이 지나면 만료**됩니다. 만료되면 새로 만들어 교체하세요.
* **기본 (프록시)** — 키 없이 본인 **프록시 IP**로 받습니다. 한도가 IP 기준이라, 지갑이 많으면 **프록시를 여러 개** 쓰는 게 좋습니다.

> 💡 **퍼블릭 민트는 둘 다 필요 없습니다.** 퍼블릭은 voucher 없이 바로 민트되므로 키도 프록시도 무관합니다. 이 선택은 **WL/서명 단계에서만** 나타납니다.

> 키가 등록돼 있으면 **OpenSea API가 기본 선택**이고, 작업을 만들 때 **기본(프록시)** 로 바꿀 수 있습니다.

## 일반 단계 (모든 WL 민트 공통)

1. **앱이 최신 버전인지 확인** (하단 왼쪽 버전).
2. **작업을 만들고 로그를 미리 확인**: 실행(Run)해서 로그 메시지가 정상인지 봅니다.
3. **민팅 3~5분 전에 중지했다가 다시 실행**: 프로젝트가 막판에 설정을 바꾸는 경우가 있어 최신 데이터를 다시 불러오기 위해서입니다.
4. **백업으로 수동 민팅 준비**: 만약 민팅이 시작됐는데 5~10초 안에 작업 로그가 "전송 대기"로 안 바뀌면, 프로젝트 사이트에서 수동 민팅하세요.

> 💡 **WL 지갑 확인**: [지갑] 화면 → 지갑 선택 → **화이트리스트 확인**으로, 어떤 지갑이 그 드롭의 WL에 들어있는지 미리 알 수 있습니다.

## 커스텀 컨트랙트 WL (고급)

런치패드가 아닌 자체 컨트랙트 WL은 보통 **서명/proof가 지갑마다 다릅니다.** 이 경우:

* 운영자가 별도 안내(모듈/hex)를 주면 그대로 따르세요.
* 또는 본인의 정확한 proof를 구해 인자/hex에 넣어야 합니다 (난이도 높음).

> ⚠️ WL 민트에서 가장 흔한 실수는 **WL에 없는 지갑으로 시도**하거나 **proof가 틀린 것**입니다. 실행 전 로그로 꼭 확인하세요.
